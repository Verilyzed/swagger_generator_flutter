# Edge-Case Test Specs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build two hand-written OpenAPI specs (3.0.3 and 3.1.0) packed with rare/edge-case constructs, characterize the generator's current output via e2e assertions and example regeneration, and record every gap in a prioritized list.

**Architecture:** Two JSON specs live in `example/lib/specs/` (single source of truth, generated by `build_runner`). A new `test/e2e/edge_cases_test.dart` loads each spec from disk and asserts the exact current generated output. Cases that produce non-compiling code or crash generation are carved out of the example and recorded in `docs/superpowers/EDGE_CASE_GAPS.md`. No generator code changes.

**Tech Stack:** Dart, `package:test`, `package:build`/`build_runner`, the project's `generateSources` entry point.

## Global Constraints

- This work changes **no** generator source under `lib/`. Only specs, tests, and docs.
- Verification bar: `dart test` green; `dart analyze` clean on package and example; `cd example && dart run build_runner build` succeeds and `dart analyze` clean. (The `--delete-conflicting-outputs` flag is ignored by the installed `build_runner`.)
- No emojis; concise, factual docs (per CLAUDE.md).
- Characterization workflow per case: add JSON -> regenerate -> read actual output -> assert it -> if it breaks compilation or crashes generation, carve the case out of the example (keep it as an e2e-only fixture or remove it) and add a gap entry. Predicted assertion values below are starting points; the actual generated string is the source of truth.
- The existing e2e test reads a spec via `File('example/lib/specs/<name>.json').readAsStringSync()` then `generateSources(content, path: <path>, baseName: <base>)` returning a `Map<String,String>` keyed by `.enums.dart`, `.models.dart`, `.service.dart`, `.client.dart`, `.api.dart`. Follow that pattern.
